# 투자 리서치 어시스턴트

**README 언어** · [English](../../README.md) · [繁體中文](README.zh-Hant.md) · [简体中文](README.zh-Hans.md) · [日本語](README.ja.md) · [Tiếng Việt](README.vi.md) · [한국어](README.ko.md)

영문 README가 주로 관리되는 원본이며, 다른 언어 파일은 읽기 편의를 위한 번역입니다.

Investment Research Agent는 내 컴퓨터에서 사용하는 개인 투자 어시스턴트입니다. 포트폴리오 추적, 거래 기록, 증권사 자료 가져오기, 보유 자산 기반 리서치와 실행 계획 작성을 도와줍니다. 권장 진입점은 로컬 웹 UI입니다. 일반 터미널에서 UI server를 직접 시작하고, 계정·보유·리포트·설정 같은 일상 작업은 브라우저 UI로 처리하며, 자연어 자동화나 분석이 필요할 때만 내장 agent CLI를 사용하세요.

## 리포트 예시

**[Report Demo](https://lpopo0856.github.io/investment-research-agent/)**

## 빠른 시작

### 준비할 것

- Codex, Claude Code 또는 이 workspace의 파일을 읽고 수정할 수 있는 다른 로컬 coding assistant.
- GitHub에서 이 repo를 설치할 때는 Git 사용을 권장합니다. Git을 모르거나 아직 설치하지 않았다면 먼저 [Git 초보자 설치 안내](git-install.ko.md)를 보세요. ZIP 다운로드도 한 번 시험해 보는 용도로는 가능하지만, 독립 복사본이므로 어시스턴트가 관리하는 업그레이드 흐름을 사용할 수 없습니다.
- 실제 포트폴리오를 만들고 싶다면 명세서, CSV, 스프레드시트, PDF, 스크린샷, 붙여넣은 거래 내역 같은 선택적 증권사 데이터.

프로젝트 내부를 배울 필요는 없습니다. 일반 터미널에서 로컬 UI를 시작한 뒤, 일상 작업은 브라우저 UI에서 직접 처리하세요. 내장 agent CLI는 복잡한 파일 가져오기, 복잡한 활동 기록, 투자 리서치, 새 리포트 생성, 유지보수처럼 자연어가 더 적합한 작업에 사용합니다.

### 시작하기

1. **Git으로 이 repo를 설치합니다(권장).** `https://github.com/lpopo0856/investment-research-agent.git`에서 clone한 뒤 폴더를 여세요. Git이 처음이라면 [Git 초보자 설치 안내](git-install.ko.md)를 따라 하세요. ZIP만 내려받아도 시험 사용은 가능하지만, 이후 업데이트는 수동으로 해야 합니다. 계속 사용할 경우 Git 설치를 권장합니다.
2. **터미널에서 로컬 UI를 시작합니다.** repo root에서 UI server를 실행해, 끝나면 닫을 수 있는 보이는 터미널 창에 server가 남도록 합니다:

   ```bash
   python3 -m venv .venv-ui
   source .venv-ui/bin/activate
   pip install -r requirements-ui.txt
   python3 scripts/run_ui_server.py
   ```

   브라우저가 자동으로 열리지 않으면 `http://127.0.0.1:8765/`를 여세요. 이후 UI에서 계정 선택/생성, 보유 확인, 기존 리포트 열기, 설정 수정과 미리보기를 직접 할 수 있습니다.

3. **먼저 UI를 쓰고, 필요할 때만 내장 agent CLI를 사용합니다.** 일반 탐색과 간단한 수정은 브라우저 컨트롤로 처리하는 것이 좋습니다. 자연어가 더 알맞은 작업은 내장 Claude/Codex agent CLI를 사용하세요. 예:

   > "이 repo를 최신 release tag로 전환한 뒤 시작을 도와줘."
   > "이 증권사 명세서를 가져오고, 저장하기 전에 무엇을 읽었는지 보여줘."
   > "내 메모에 있는 이 거래들을 기록해줘."
   > "demo 데이터로 포트폴리오 리포트를 만들어줘."
   > "현재 포트폴리오 기준으로 NVDA를 분석해줘."

   첫 문장으로 위의 release tag 전환을 권장합니다. 포트폴리오 데이터가 기록되기 전에 최신 릴리스로 맞춰 두는 안전 장치이며, Git 터미널이든 GitHub Desktop이든 같은 문장이 동작합니다.

4. **저장 전에 검토합니다.** 어시스턴트는 포트폴리오 데이터를 쓰기 전에 설정, 거래, 가져오기 변경안을 미리 보여주므로 먼저 확인하거나 수정할 수 있습니다.

## 이미 ZIP을 다운로드한 경우

ZIP 다운로드는 빠른 demo 시험에는 괜찮지만 독립 복사본입니다. Git history가 없기 때문에 어시스턴트가 branch나 release tag를 안전하게 맞춰 자동 업그레이드할 수 없습니다.

장기 사용을 원한다면:

1. 개인 포트폴리오 파일이 안전한지 확인하기 전까지 ZIP 복사본을 보관하세요.
2. Git 또는 GitHub Desktop으로 repo를 다시 설치하세요.
3. ZIP 복사본에 실제 포트폴리오 데이터가 들어 있다면 삭제하지 마세요. 그 폴더를 어시스턴트로 열고 “이 ZIP 설치본을 Git 설치본으로 안전하게 옮겨줘.”라고 말하세요.

## 업그레이드

이미 로컬에 있는 repo를 업데이트하려면 이 폴더를 어시스턴트로 열고 이렇게 요청하세요:

> "이 repo를 안전하게 업그레이드해줘."

어시스턴트는 `upgrade-management` skill을 사용해 필요할 때 개인 포트폴리오 데이터를 백업하고, Git으로 설치된 repo 코드와 의존성을 업데이트하며, 계정 레이아웃을 확인합니다. migration 또는 포트폴리오 데이터 쓰기가 필요하면 실행 전에 확인을 받습니다. Git 설치본은 현재 branch 또는 release tag를 안전하게 따라갈 수 있습니다. ZIP/archive 버전은 독립 복사본으로 취급하므로 자동 업데이트하지 않고, 수동 업데이트 방법을 안내하며 이후에는 Git 사용을 권장합니다.

## 주요 기능

기술적인 세부 지식은 필요하지 않습니다. 로컬 UI를 홈베이스로 사용하세요. 계정, 보유, 리포트, 설정은 직접 클릭해 다루고, 자연어 자동화나 분석이 필요할 때 내장 agent CLI를 사용합니다. 저장된 포트폴리오 데이터를 바꾸는 작업은 UI/agent 흐름이 저장 전에 변경안을 보여 주고 확인을 받은 뒤 진행합니다.

- **시작하기** — 첫 계정을 만들고, 초기 포트폴리오를 가져오며, 투자 스타일을 설정합니다.
- **투자 전략 설정** — 위험 감수 성향, 포지션 규모, 보유 기간, 언어, 기준 통화, 손대지 않을 영역을 정합니다.
- **여러 계정 관리** — 증권 계좌, 연금 계좌, 지역별 포트폴리오, 전략별 계정을 만들고 전환·확인·합산할 수 있습니다.
- **활동 기록** — 매수, 매도, 입금, 출금, 배당, 수수료, 환전을 평소 표현으로 추가합니다.
- **증권사 파일 가져오기** — 명세서, 내보내기 파일, 스프레드시트, PDF, 스크린샷, 붙여넣은 거래 내역을 가져옵니다.
- **기록 수정** — 잘못된 거래, 중복 항목, 현금 잔액, 미청산 로트를 증권사 자료와 대조해 수정합니다.
- **투자 리서치** — 주식, ETF, 테마, 섹터, 시장, 포트폴리오 익스포저를 본인의 전략에 맞춰 분석합니다.
- **일일 리포트** — 가격, 뉴스, 이벤트, 경고, 기회, 조정 제안, 오늘의 액션 리스트를 담은 당일 의사결정 대시보드를 만듭니다.
- **포트폴리오 리포트** — 일중 매매 노이즈를 제외하고 보유, 현금, 배분, 손익, 집중도, 투자 속도, 구조를 점검합니다.
- **종합 리포트** — 각 계정의 기록은 분리한 채 모든 계정을 합친 상위 뷰를 만듭니다.
- **위험 및 익스포저 점검** — 과도하게 집중되었거나, 겹치거나, 위험이 높거나, 자금이 부족하거나, 우선 확인이 필요한 부분을 물어볼 수 있습니다.

## 자세한 사용법

### 로컬 UI 열기

이 프로젝트를 매일 쓰기에는 로컬 UI가 가장 쉽습니다. repo root의 보이는 터미널에서 시작하세요:

```bash
source .venv-ui/bin/activate  # 위에서 venv를 만들었다면
python3 scripts/run_ui_server.py
```

그런 다음 `http://127.0.0.1:8765/`를 엽니다. 브라우저 대시보드가 일반 사용의 시작점입니다. 일상 작업은 직접 UI 컨트롤을 우선 사용하세요:

- 계정 선택, 생성, 전환
- 보유 확인과 빠른 작업
- 기존 리포트 열기와 리포트 뷰어
- 설정을 섹션별로 수정하고 미리보기
- 자연어 도움이 필요한 작업에서만 내장 Claude/Codex agent CLI 실행

내장 agent CLI는 클릭보다 글로 설명하기 쉬운 작업에 적합합니다. 메모에서 거래 기록하기, 증권사 파일 가져오기, 복잡한 기록 대조하기, 보유 종목 리서치하기, 새 리포트 만들기, repo 업그레이드/유지보수 등이 해당됩니다. 설정 또는 ledger 데이터를 저장하기 전에는 동일한 확인 gate가 적용됩니다.

### 무엇을 할 수 있는지 묻기

어디서 시작할지 모르겠다면 먼저 UI의 계정, 보유, 리포트, 설정 탭을 살펴보세요. 그래도 안내가 필요하면 내장 agent CLI에 물어보세요:

> "여기서 뭘 할 수 있어?"
> "가능한 기능을 보여 줘."
> "도움말."

내장 agent가 시작 설정, 거래 기록, 투자 리서치, 리포트 작성 메뉴를 간단히 보여 줍니다.

### 시작하거나 포트폴리오 만들기

처음 설정할 때 단순히 계정을 만들거나 선택하는 일은 UI에서 직접 할 수 있습니다. 안내가 필요하거나 증권사 데이터를 가져와야 한다면 내장 agent CLI에 목표를 설명하세요.

> "시작하는 것 도와줘."
> "온보딩해 줘."
> "이 명세서로 포트폴리오를 만들어 줘."
> "이 증권사 파일을 가져와서 포트폴리오를 설정해 줘."

증권사 명세서, 스프레드시트, PDF, 스크린샷, 붙여넣은 거래 내역을 첨부할 수 있습니다. 어시스턴트는 계정 생성, 투자 스타일 파악, 기록 준비, 설정 확인을 도와줍니다.

### 투자 스타일 설정 또는 검토

간단한 설정 변경은 UI의 설정 편집기에서 섹션별로 직접 수정하고 미리볼 수 있습니다. 전략 문구 정리, 일관성 검토, 여러 관련 설정의 한 번에 변경이 필요하면 내장 agent CLI에 말하세요.

> "설정을 같이 보자."
> "내 투자 전략을 검토해 줘."
> "위험 감수 성향을 업데이트해 줘."
> "기준 통화를 원화로 바꿔 줘."
> "리포트는 한국어로 써 줘."
> "이 섹터들은 투자하지 않을 거야."

위험 감수 성향, 포지션 규모, 보유 기간, 진입 원칙, 선호 언어, 기준 통화, 피하고 싶은 영역을 포함할 수 있습니다.

### 계정 추가 및 관리

여기서 "계정"은 따로 기록하는 장부일 뿐입니다. 가족별(본인, 배우자, 자녀 교육비), 목표별(은퇴, 주택, 비상금), 전략별(코어, 위성, 실험 포지션), 세제별(ISA, 연금저축, 일반 위탁), 또는 익숙하다면 시장별(코스피·코스닥, 미국, 일본)로 자유롭게 나눌 수 있습니다. 분할 방식은 도구가 강제하지 않습니다.

일반적인 생성, 목록 보기, 전환은 UI의 계정 컨트롤로 직접 하는 것을 권장합니다. 계정 설정에 설명이나 여러 단계가 필요하거나, 선택한 계정에서 리포트/액션을 만들어야 할 때만 내장 agent CLI를 사용하세요.

> "위성 전략 계정을 만들어 줘(고위험 소형 포지션용)."
> "일본 포트폴리오용 계정을 새로 만들어 줘."
> "연금 계정 리포트를 만들어 줘."
> "모든 계정을 합쳐 종합 리포트를 만들어 줘."

각 계정은 설정, 거래, 현금, 보유, 리포트를 따로 가집니다. 종합 리포트는 상위 뷰를 만들지만 원래 계정 데이터는 분리해 둡니다.

### 거래와 현금 흐름 기록

투자 활동을 평소 말로 설명합니다.

> "어제 NVDA 30주를 185달러에 샀어."
> "오늘 TSLA 10주를 400달러에 팔았어."
> "GOOG 1분기 배당 80달러."
> "5,000달러 입금했어."
> "세금 때문에 1,000달러 출금했어."
> "거래 수수료 12달러를 냈어."
> "USD 2,000을 원화로 환전했어."

어시스턴트는 내용을 해석하고 추가할 기록을 보여 준 뒤, 확인을 받은 후 포트폴리오를 업데이트합니다.

### 기록 수정 또는 대조

잘못된 데이터가 있으면 무엇을 고칠지 설명합니다.

> "작년 9월 GOOG 로트를 수정해 줘."
> "그 NVDA 거래는 30주가 아니라 20주였어."
> "중복 배당 기록을 삭제해 줘."
> "이 명세서로 현금 잔액을 대조해 줘."
> "미청산 로트가 증권사 자료와 맞는지 확인해 줘."

어시스턴트는 저장 전에 변경안을 설명합니다.

### 증권사 파일 가져오기

파일을 첨부하고 원하는 일을 말합니다.

> "이건 증권사 내보내기 파일이야. 가져와 줘."
> "이 거래 내역을 가져와 줘."
> "이 PDF 명세서를 읽고 저장 전에 무엇을 찾았는지 보여 줘."
> "이 파일에는 배당과 거래가 있어. 내 계정에 추가해 줘."

가져오기 팁:

- PDF에 비밀번호가 있으면 직접 열어 비밀번호 없는 사본으로 저장한 뒤 가져오세요.
- 파일이 매우 크면, 특히 PDF는 더 작은 묶음으로 나누세요.
- 해석이 애매한 내용이 있으면 어시스턴트가 저장 전에 확인을 요청합니다.

### 주식, ETF, 테마, 시장 리서치

포트폴리오에 맞춘 리서치를 바로 요청할 수 있습니다.

> "현재 포트폴리오 기준으로 NVDA를 분석해 줘."
> "내 전략상 지금 TSM을 사야 할까?"
> "TSM, NVDA, AMD를 비교해 줘."
> "지금 내 AI 익스포저가 얼마나 돼?"
> "실적 발표 전에 단기 포지션을 줄여야 할까?"
> "일본 익스포저를 검토해 줘."
> "이번 주 반도체에서 무엇을 봐야 해?"

리서치는 매수, 보유, 축소, 회피, 대기 중 어떤 결정을 할지, 적절한 규모는 얼마인지, 판단이 틀렸다고 볼 조건은 무엇인지, 다음에 추적할 변수는 무엇인지에 집중합니다.

### 일일 리포트 만들기

일일 리포트는 오늘의 의사결정 대시보드가 필요할 때 사용합니다.

> "오늘 일일 리포트 만들어 줘."
> "장 시작 전 리포트 돌려 줘."
> "오늘 포트폴리오 건강 점검을 보여 줘."
> "오늘 포트폴리오에서 뭘 해야 해?"

일일 리포트에는 최신 가격, 포트폴리오 상태, 주요 뉴스, 예정 이벤트, 위험 경고, 좋은 기회, 조정 제안, 명확한 오늘의 액션 리스트가 포함될 수 있습니다.

### 포트폴리오 리포트 만들기

포트폴리오 리포트는 일중 노이즈가 아니라 구조를 보고 싶을 때 사용합니다.

> "내 포트폴리오 리포트를 만들어 줘."
> "배분과 성과를 보여 줘."
> "보유, 현금, 집중도를 검토해 줘."
> "내 포트폴리오 성과는 어때?"

포트폴리오 리포트는 보유, 현금, 배분, 손익, 집중도, 투자 속도, 구조에 집중합니다. 당일 매매 결정을 만들기보다 전체 장부의 모양을 점검하는 데 적합합니다.

### 여러 계정의 종합 리포트 만들기

계정이 여러 개라면 합산 뷰를 요청할 수 있습니다.

> "모든 계정을 합쳐 종합 리포트를 만들어 줘."
> "전체 포트폴리오를 보여 줘."
> "모든 계정을 합산해서 익스포저를 정리해 줘."

종합 리포트는 여러 계정을 아우르는 상위 뷰를 제공하며, 특정 계정에 대한 매매 제안은 피합니다.

### 위험과 익스포저 묻기

전체 리포트를 만들지 않아도 집중 질문을 할 수 있습니다.

> "내 가장 큰 집중 위험은 뭐야?"
> "현금은 얼마나 있어?"
> "어떤 포지션이 가장 많이 겹쳐?"
> "AI 반도체에 너무 몰려 있어?"
> "지금 가장 위험한 보유 종목은 뭐야?"
> "앞으로 어떤 이벤트가 포트폴리오에 영향을 줘?"

### 실행 계획 요청하기

짧은 결정 목록만 원한다면 이렇게 말합니다.

> "오늘 액션 리스트를 줘."
> "매수, 축소, 그대로 두기 중 뭐가 맞아?"
> "어떤 포지션을 먼저 봐야 해?"
> "이번 주 무엇을 추적해야 하는지 알려 줘."

명확한 우위가 없을 때 어시스턴트는 억지 거래보다 아무것도 하지 않는 결정을 우선합니다.
