Investment Research Agent 是一個在本機使用的私人投資助理，可協助你追蹤投資組合、記錄交易、匯入券商資料，並把持股轉成清楚的研究與行動計畫。開啟工作區後，用平常說話的方式描述你想做什麼即可。

## 快速開始

### 你需要準備什麼

- 一個本機 coding assistant，例如 Codex、Claude Code，或其他能讀寫這個 workspace 檔案的助理。
- 建議用 Git 從 GitHub 安裝這個 repo。如果你不會 Git 或還沒安裝，請先看 [Git 新手安裝指南](git-install.zh-Hant.md)。ZIP 下載也能用，但它是獨立副本，不能使用助理管理的自動升級流程。
- 如果要建立真實投資組合，可準備選用的券商資料，例如對帳單、CSV、試算表、PDF、截圖或貼上的交易紀錄。

你不需要學專案腳本，也不需要手動安裝技術套件。Onboarding 時，助理會檢查本機環境、視需要安裝少量依賴，並告訴你是否有需要處理的事項。

### 開始使用

1. **用 Git 安裝這個 repo（建議）。** 從 `https://github.com/lpopo0856/investment-research-agent.git` clone，然後開啟資料夾。如果 Git 對你來說很陌生，請照著 [Git 新手安裝指南](git-install.zh-Hant.md) 操作。如果你只是下載 ZIP，也可以先試用，但之後更新需要手動處理；長期使用建議改用 Git 安裝。
2. **用你的助理開啟這個資料夾。** 使用能處理這個 workspace 檔案的本機助理。
3. **用一般說話方式開始。** 適合一開始這樣說：

   > 「請把這個 repo 切換到最新的 release tag，然後幫我開始。」
   > 「建立我的第一個投資帳戶。」
   > 「匯入這份券商對帳單，儲存前先告訴我你讀到了什麼。」
   > 「用 demo 資料產出投組報告。」

   建議第一句先用上面的 release tag 切換，可以確保在任何投資資料寫入前已經在最新版本上。不論你是用 Git 終端機還是 GitHub Desktop 安裝，這句話都適用。

4. **儲存前先檢查。** 助理在寫入投資資料前，會先預覽設定、交易與匯入變更，讓你先確認或修改方案。

## 如果你已經下載 ZIP

ZIP 下載可以用來快速試用 demo，但它是獨立副本，沒有 Git history，所以助理無法安全對齊 branch 或 release tag 來自動升級。

長期使用建議：

1. 在確認私人投資資料安全前，先保留你的 ZIP 副本。
2. 改用 Git 或 GitHub Desktop 重新安裝 repo。
3. 如果 ZIP 副本已經包含真實投資資料，請不要刪掉它。用助理開啟該資料夾並說：「請幫我把這個 ZIP 安裝安全搬到 Git 安裝。」

## 升級

如果要更新既有的本機 repo，請用助理開啟這個資料夾並說：

> 「請幫我安全升級這個 repo。」

助理會使用 `upgrade-management` skill：需要時先備份私人投資資料、更新 Git 安裝的 repo 程式碼與依賴、檢查帳戶版面，並在任何 migration 或投資資料寫入前停下來請你確認。Git 安裝可以安全跟隨目前 branch 或 release tag；ZIP/archive 版本會被視為獨立副本，助理只會說明手動更新步驟，並建議未來改用 Git 升級。

## 主要功能

你不需要懂技術細節。直接用自然語言描述意圖，助理會處理流程。只要會改動已儲存的投資資料，助理都會先列出預計變更，等你確認後才會儲存。

- **開始使用** — 建立第一個帳戶、匯入初始投資組合，並設定你的投資風格。
- **設定投資策略** — 定義風險承受度、部位大小、持有期間、語言、基準幣別與不碰的領域。
- **管理多個帳戶** — 建立、切換、檢視與合併多個券商帳戶、退休帳戶、區域投資組合或策略帳戶。
- **記錄活動** — 用一般說法新增買進、賣出、入金、出金、股息、手續費與換匯。
- **匯入券商資料** — 匯入對帳單、匯出檔、試算表、PDF、截圖或貼上的交易紀錄。
- **修正紀錄** — 更正錯誤交易、移除重複項目、核對現金，並用券商資料檢查未平倉批次。
- **研究投資標的** — 依照你的策略分析股票、ETF、主題、產業、市場與投組曝險。
- **每日報告** — 產出當天決策儀表板，包含價格、新聞、事件、警示、機會、建議調整與今日行動清單。
- **投組報告** — 檢視持股、現金、配置、損益、集中度、節奏與投組結構，排除日內交易雜訊。
- **總報告** — 合併所有帳戶成為高階總覽，同時保留各帳戶各自的資料。
- **風險與曝險檢查** — 詢問哪些部位過度集中、重疊、高風險、資金不足或需要優先處理。

## 詳細用法

### 詢問可以做什麼

如果不確定從哪裡開始，可以這樣說：

> 「這裡能做什麼？」
> 「幫我看有哪些功能。」
> 「Help。」

助理會給你一個簡單選單，包含 onboarding、記錄交易、投資研究與產出報表。

### 開始使用或建立投資組合

第一次設定時，直接描述目標：

> 「幫我開始。」
> 「帶我 onboarding。」
> 「請用這份對帳單幫我建立投組。」
> 「匯入這份券商檔案，並幫我設定投資組合。」

你可以附上券商對帳單、試算表、PDF、截圖或貼上的交易紀錄。助理會協助建立帳戶、理解你的投資風格、準備紀錄，並確認設定可正常使用。

### 設定或檢視投資風格

告訴助理你怎麼投資，讓後續研究用你的語氣與紀律做判斷：

> 「帶我過一遍設定。」
> 「檢視我的投資策略。」
> 「更新我的風險承受度。」
> 「把我的基準幣別改成 TWD。」
> 「報表使用繁體中文。」
> 「這些產業我不碰。」

這可以包含風險承受度、部位大小、持有期間、進場紀律、偏好語言、基準幣別與避開的領域。

### 新增與管理帳戶

這裡的「帳戶」就是分開記的帳本，怎麼切都行。常見分法：依家人（自己、另一半、小孩教育金）、依目的（退休、買房、緊急預備金）、依策略（核心、衛星、試驗倉）、依稅務分類（一般、退休帳戶），或如果你習慣，也可以依股票市場（台股、美股、日股）切。工具不限定分法。

> 「新增一個退休帳戶。」
> 「幫我另一半開一個帳戶。」
> 「新增一個小孩教育金帳本。」
> 「新增一個衛星策略帳戶（高風險小部位）。」
> 「新增一個日本投資組合帳戶。」
> 「列出我的所有帳戶。」
> 「我現在使用哪個帳戶？」
> 「切換到預設帳戶。」
> 「切換到台股帳戶。」
> 「產出退休帳戶的報表。」
> 「把所有帳戶合併產出總報告。」

每個帳戶都有自己的設定、交易、現金、持股與報表。總報告會把帳戶合併成高階總覽，但底層資料仍分開保存。

### 記錄交易與現金流

用日常說法描述投資活動：

> 「昨天用 185 美元買了 30 股 NVDA。」
> 「今天用 400 美元賣掉 10 股 TSLA。」
> 「GOOG 第一季股息 80 美元。」
> 「入金 5,000 美元。」
> 「為了繳稅出金 1,000 美元。」
> 「我付了 12 美元手續費。」
> 「我把 USD 2,000 換成 TWD。」

助理會解析內容、顯示預計新增的紀錄，並等你確認後才更新投組。

### 修正或核對紀錄

如果資料有錯，直接描述要修正什麼：

> 「修正去年九月那筆 GOOG 批次。」
> 「那筆 NVDA 應該是 20 股，不是 30 股。」
> 「移除重複的股息紀錄。」
> 「用這份對帳單核對現金餘額。」
> 「檢查我的未平倉批次是否和券商資料一致。」

助理會先說明預計修改內容，再等你確認後儲存。

### 匯入券商資料

附上檔案後描述需求：

> 「這是我的券商匯出檔，請幫我匯入。」
> 「匯入這份交易紀錄。」
> 「匯入這份 PDF 對帳單，儲存前先告訴我你讀到了什麼。」
> 「這份檔案有股息和交易，請加到我的帳戶。」

匯入小提示：

- 如果你持有台股，券商（元大、富邦、群益、永豐金、玉山、凱基、國泰等）匯出檔或台灣證券交易所匯出檔通常是最佳來源。海外複委託與美股，多半也能直接從同一家券商匯出。
- 如果 PDF 有密碼，請先自行開啟並另存成無密碼版本再匯入。
- 如果檔案很大，尤其是 PDF，請拆成較小批次再匯入。
- 如果匯入內容有歧義，助理會請你確認解讀後才儲存。

### 研究股票、ETF、主題或市場

可以直接問投組相關研究：

> 「對照我現在的投組分析 NVDA。」
> 「以我的策略來看，現在該買 TSM 嗎？」
> 「比較 TSM、NVDA、AMD。」
> 「我現在 AI 曝險有多高？」
> 「財報前要不要調降短線部位？」
> 「檢視我的日本曝險。」
> 「這週半導體我該注意什麼？」

研究筆記會聚焦決策：買進、持有、減碼、避開或等待；部位該多大；什麼情況代表判斷錯了；接下來追蹤哪些變數。

### 產出每日報告

每日報告適合當天決策使用：

> 「產出今天的每日報告。」
> 「跑我的盤前報告。」
> 「給我今天的投組健檢。」
> 「今天投組我該做什麼？」

每日報告可以包含最新價格、投組健康狀態、重要新聞、即將到來的事件、風險警示、高機會設定、建議調整與明確的今日行動清單。

### 產出投組報告

投組報告適合檢視結構，不追逐日內雜訊：

> 「產出我的投組報告。」
> 「顯示我的配置與績效。」
> 「檢視我的持股、現金與集中度。」
> 「我的投組表現如何？」

投組報告聚焦持股、現金、配置、損益、集中度、投資節奏與投組結構，適合檢視整本帳，而不是產生當天交易建議。

### 產出跨帳戶總報告

有多個帳戶時，可以要求合併視角：

> 「把所有帳戶合併產出總報告。」
> 「顯示我的整體投資組合。」
> 「合併所有帳戶並整理我的曝險。」

總報告提供跨帳戶的高階總覽，並避免對單一帳戶產生交易建議。

### 詢問風險與曝險

不產出完整報表，也可以問聚焦問題：

> 「我最大的集中風險是什麼？」
> 「我現在有多少現金？」
> 「哪些部位重疊最多？」
> 「我是不是太集中在 AI 半導體？」
> 「哪些持股現在風險最高？」
> 「接下來哪些事件會影響我的投組？」

### 要一份行動計畫

如果只想要短清單，可以說：

> 「給我今天的行動清單。」
> 「我該買進、減碼，還是什麼都不做？」
> 「哪些部位要優先處理？」
> 「告訴我這週要追蹤什麼。」

沒有明確優勢時，助理會偏向不行動，而不是硬湊交易。

## 隱私

你的設定、交易紀錄與產出的報表都留在這個本機工作區。這個專案不會替你公開資料。你可以控制要分享哪些檔案給助理，以及哪些變更要儲存。

## 第三方市場資料

報表與研究可能使用公開行情與匯率來源，以及你自行提供的資料存取。資料可用性、延遲、使用限制與準確性取決於各資料提供者。

## 免責聲明

僅供個人研究與紀錄使用，不構成投資、法律或稅務建議。交易前請自行查證重要事實；決策與結果由你自行負責。
