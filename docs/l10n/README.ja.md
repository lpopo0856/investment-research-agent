Investment Research Agent は、ポートフォリオの追跡、取引記録、証券会社の明細取り込み、保有銘柄にもとづくリサーチと行動計画づくりを助ける、ローカルで使うプライベートな投資アシスタントです。ワークスペースを開いたら、ふだんの言葉でやりたいことを伝えてください。

## クイックスタート

### 用意するもの

- Codex、Claude Code、またはこの workspace のファイルを読み書きできるローカル coding assistant。
- GitHub からこの repo を入手するには Git でのインストールをおすすめします。Git が初めて、または未インストールの場合は、まず [Git 初心者向けインストールガイド](git-install.ja.md) を見てください。ZIP ダウンロードでも一度試すことはできますが、独立したコピーなのでアシスタント管理のアップグレードは使えません。
- 実際のポートフォリオを作る場合は、任意で証券会社データを用意します。明細、CSV、表計算ファイル、PDF、スクリーンショット、貼り付けた取引履歴などです。

プロジェクトのスクリプトを覚えたり、技術パッケージを手でインストールしたりする必要はありません。Onboarding 中に、アシスタントがローカル環境を確認し、必要なら少数の依存パッケージを入れ、対応が必要なことを知らせます。

### 使い始める

1. **Git でこの repo をインストールします（推奨）。** `https://github.com/lpopo0856/investment-research-agent.git` から clone して、フォルダを開きます。Git が初めての場合は [Git 初心者向けインストールガイド](git-install.ja.md) に沿って進めてください。ZIP だけをダウンロードした場合も試用はできますが、今後の更新は手動になります。継続利用には Git でのインストールをおすすめします。
2. **アシスタントでこのフォルダを開きます。** この workspace のファイルを扱えるローカルアシスタントを使います。
3. **ふだんの言葉で始めます。** 最初は、たとえば次のように話します：

   > 「この repo を最新の release tag に切り替えて、はじめるのを手伝ってください。」
   > 「最初の投資口座を作って。」
   > 「この証券会社の明細を取り込んで、保存前に読み取った内容を見せて。」
   > 「demo データでポートフォリオレポートを作って。」

   最初の一言は上の release tag 切り替えから始めるのがおすすめです。ポートフォリオデータに何か書き込まれる前に最新リリースに揃えられます。Git ターミナルでも GitHub Desktop でも、同じ言い方で動きます。

4. **保存前に確認します。** 投資データを書き込む前に、アシスタントは設定、取引、取り込み内容の変更案をプレビューするので、先に確認または修正できます。

## すでに ZIP をダウンロードした場合

ZIP ダウンロードは demo を試すには使えますが、独立したコピーです。Git history がないため、アシスタントは branch や release tag を安全に合わせて自動アップグレードできません。

継続利用する場合：

1. 非公開ポートフォリオファイルが安全だと確認できるまで、ZIP コピーを残してください。
2. Git または GitHub Desktop で repo をインストールし直します。
3. ZIP コピーに実際のポートフォリオデータが入っている場合は削除しないでください。そのフォルダをアシスタントで開き、「この ZIP インストールを Git インストールへ安全に移すのを手伝って。」と伝えてください。

## アップグレード

既存のローカル repo を更新したい場合は、このフォルダをアシスタントで開いて、次のように依頼してください：

> 「この repo を安全にアップグレードしてください。」

アシスタントは `upgrade-management` skill を使い、必要に応じて非公開ポートフォリオデータをバックアップし、Git でインストールされた repo のコードと依存パッケージを更新し、口座レイアウトを確認します。migration やポートフォリオデータの書き込みが必要な場合は、実行前に確認します。Git インストールでは現在の branch または release tag を安全に追跡できます。ZIP/archive 版は独立コピーとして扱い、自動更新せず、手動更新手順を案内して今後は Git をすすめます。

## 主な機能

技術的な詳しい知識は必要ありません。自然な言葉で意図を伝えれば、アシスタントが流れを処理します。保存済みのポートフォリオデータを変更する場合は、保存前に変更案を見せて確認を求めます。

- **はじめる** — 最初の口座を作成し、初期ポートフォリオを取り込み、投資スタイルを設定します。
- **投資戦略を設定する** — リスク許容度、ポジションサイズ、保有期間、言語、基準通貨、触らない領域を定義します。
- **複数口座を管理する** — 証券口座、退職金口座、地域別ポートフォリオ、戦略別口座を作成・切替・確認・合算できます。
- **活動を記録する** — 買付、売却、入金、出金、配当、手数料、為替交換をふだんの表現で追加できます。
- **証券会社ファイルを取り込む** — 明細、エクスポート、表計算ファイル、PDF、スクリーンショット、貼り付けた取引履歴を取り込めます。
- **記録を修正する** — 誤った取引、重複、現金残高、未決済ロットを証券会社の明細と照合できます。
- **投資対象をリサーチする** — 株式、ETF、テーマ、セクター、市場、ポートフォリオ・エクスポージャーを自分の戦略に沿って分析できます。
- **デイリーレポート** — 価格、ニュース、イベント、アラート、機会、調整案、今日のアクションリストを含む当日の意思決定ダッシュボードを作れます。
- **ポートフォリオレポート** — 日々の売買ノイズを除き、保有、現金、配分、損益、集中度、ペース、構造を確認できます。
- **合算レポート** — 各口座の記録は分けたまま、全口座を合算した上位ビューを作れます。
- **リスクとエクスポージャー確認** — 集中、重複、高リスク、資金不足、優先対応が必要な箇所を質問できます。

## 詳しい使い方

### 何ができるか聞く

どこから始めるか迷ったら、こう伝えてください。

> 「ここで何ができる？」
> 「できることを見せて。」
> 「ヘルプ。」

アシスタントは、初期設定、取引記録、投資リサーチ、レポート作成の簡単なメニューを返します。

### はじめる、またはポートフォリオを作る

新しく設定するときは、目的をそのまま伝えます。

> 「始めるのを手伝って。」
> 「オンボーディングして。」
> 「この明細でポートフォリオを作って。」
> 「この証券会社ファイルを取り込んで、ポートフォリオを設定して。」

証券会社の明細、表計算ファイル、PDF、スクリーンショット、貼り付けた取引履歴を添付できます。アシスタントは口座作成、投資スタイルの把握、記録準備、セットアップ確認を手伝います。

### 投資スタイルを設定・確認する

今後のリサーチが自分の考え方と規律に沿うよう、投資のしかたを伝えます。

> 「設定を一緒に見て。」
> 「投資戦略を確認して。」
> 「リスク許容度を更新して。」
> 「基準通貨を円にして。」
> 「レポートは日本語で。」
> 「このセクターには投資しない。」

リスク許容度、ポジションサイズ、保有期間、エントリー規律、使用言語、基準通貨、避けたい領域を含められます。

### 口座を追加・管理する

ここでの「口座」は別の帳簿のことです。区分の仕方は自由——家族別（自分・配偶者・子どもの教育資金）、目的別（老後、住宅、緊急予備）、戦略別（コア、サテライト、試験ポジション）、税制別（特定口座、NISA、iDeCo）、もしくは慣れた人なら市場別（日本株、米国株、台湾株）でも構いません。本ツールが分け方を決めることはありません。

> 「退職金用の口座を追加して。」
> 「配偶者の口座を作って。」
> 「子どもの教育資金の帳簿を追加して。」
> 「サテライト戦略の口座を追加して（高リスク・小ポジション）。」
> 「米国株ポートフォリオ用の口座を追加して。」
> 「すべての口座を見せて。」
> 「今どの口座を使っている？」
> 「標準口座に切り替えて。」
> 「NISA 口座に切り替えて。」
> 「退職金口座のレポートを作って。」
> 「全口座を合算して合算レポートを作って。」

各口座は設定、取引、現金、保有、レポートを別々に持ちます。合算レポートは上位ビューを作りますが、元の口座データは分けて保ちます。

### 取引とキャッシュフローを記録する

投資活動をふだんの言葉で説明します。

> 「昨日 NVDA を 185 ドルで 30 株買った。」
> 「今日 TSLA を 400 ドルで 10 株売った。」
> 「GOOG の第1四半期配当、80ドル。」
> 「5,000ドルを入金した。」
> 「税金用に1,000ドルを出金した。」
> 「12ドルの取引手数料を払った。」
> 「USD 2,000 を円に替えた。」

アシスタントは内容を読み取り、追加予定の記録を表示し、確認後にポートフォリオを更新します。

### 記録を修正・照合する

間違いがある場合は、修正したい内容を伝えます。

> 「去年9月の GOOG ロットを修正して。」
> 「あの NVDA 取引は30株ではなく20株だった。」
> 「重複した配当記録を削除して。」
> 「この明細で現金残高を照合して。」
> 「未決済ロットが証券会社の明細と一致するか確認して。」

アシスタントは保存前に変更案を説明します。

### 証券会社ファイルを取り込む

ファイルを添付して、やりたいことを伝えます。

> 「これは証券会社のエクスポートです。取り込んで。」
> 「この取引履歴を取り込んで。」
> 「この PDF 明細を読み、保存前に内容を見せて。」
> 「このファイルには配当と取引があるので、口座に追加して。」

取り込みのコツ：

- PDF にパスワードがある場合は、自分で開いてパスワードなしのコピーを保存してから取り込んでください。
- ファイルが大きい場合、特に PDF は小さなまとまりに分けてください。
- 解釈があいまいな場合、アシスタントは保存前に確認を求めます。

### 株式、ETF、テーマ、市場をリサーチする

ポートフォリオに沿ったリサーチをそのまま頼めます。

> 「今のポートフォリオと照らして NVDA を分析して。」
> 「自分の戦略では今 TSM を買うべき？」
> 「TSM、NVDA、AMD を比較して。」
> 「今の AI エクスポージャーはどれくらい？」
> 「決算前に短期ポジションを減らすべき？」
> 「米国エクスポージャーを確認して。」
> 「今週の半導体で見るべき点は？」

リサーチは、買う・持つ・減らす・避ける・待つの判断、適切なサイズ、見立てが外れたと判断する条件、次に追うべき変数に焦点を当てます。

### デイリーレポートを作る

デイリーレポートは当日の意思決定に使います。

> 「今日のデイリーレポートを作って。」
> 「寄り付き前のレポートを作って。」
> 「今日のポートフォリオ健康診断を出して。」
> 「今日ポートフォリオで何をすべき？」

デイリーレポートには、最新価格、ポートフォリオの状態、重要ニュース、今後のイベント、リスクアラート、高い機会、調整案、明確な今日のアクションリストを含められます。

### ポートフォリオレポートを作る

ポートフォリオレポートは、日々のノイズではなく構造を確認したいときに使います。

> 「ポートフォリオレポートを作って。」
> 「配分とパフォーマンスを見せて。」
> 「保有、現金、集中度を確認して。」
> 「ポートフォリオの成績はどう？」

ポートフォリオレポートは、保有、現金、配分、損益、集中度、投資ペース、構造に焦点を当てます。当日の売買指示を作るためではなく、全体像を確認するために向いています。

### 全口座の合算レポートを作る

複数口座がある場合は、合算ビューを頼めます。

> 「全口座を合算してレポートを作って。」
> 「全体のポートフォリオを見せて。」
> 「全口座をまとめてエクスポージャーを整理して。」

合算レポートは口座をまたいだ上位ビューを提供し、特定口座向けの売買提案は避けます。

### リスクとエクスポージャーを聞く

完全なレポートを作らなくても、絞った質問ができます。

> 「最大の集中リスクは何？」
> 「現金はいくらある？」
> 「どのポジションが一番重なっている？」
> 「AI 半導体に偏りすぎている？」
> 「今リスクが高い保有銘柄は？」
> 「今後どのイベントがポートフォリオに影響する？」

### アクションプランを頼む

短い意思決定リストだけ欲しい場合は、こう伝えます。

> 「今日のアクションリストを出して。」
> 「買う、減らす、何もしないのどれ？」
> 「どのポジションを先に見るべき？」
> 「今週追うべきことを教えて。」

明確な優位性がない場合、アシスタントは無理に取引を作らず、何もしない判断を優先します。

## プライバシー

設定、取引記録、作成されたレポートはこのローカルワークスペースに残ります。このプロジェクトがあなたのデータを公開することはありません。どのファイルをアシスタントに渡し、どの変更を保存するかはあなたが管理します。

## 第三者の市場データ

レポートとリサーチでは、公開されている市場データや為替データ、あなたが提供するデータアクセスを使うことがあります。利用可否、遅延、制限、正確性は各提供元に依存します。

## 免責事項

個人のリサーチと記録管理のためのものです。投資、法務、税務の助言ではありません。取引前に重要な事実を自分で確認してください。判断と結果はあなた自身の責任です。
