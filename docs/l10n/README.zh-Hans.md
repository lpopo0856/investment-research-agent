Investment Research Agent 是一个在本地使用的私人投资助手，可以帮你跟踪投资组合、记录交易、导入券商资料，并把持仓转成清晰的研究和行动计划。打开工作区后，用平常说话的方式描述你想做什么即可。

## 快速开始

### 你需要准备什么

- 一个本地 coding assistant，例如 Codex、Claude Code，或其他能读写这个 workspace 文件的助手。
- 建议用 Git 从 GitHub 安装这个 repo。如果你不会 Git 或还没安装，请先看 [Git 新手安装指南](git-install.zh-Hans.md)。ZIP 下载也能用，但它是独立副本，不能使用助手管理的自动升级流程。
- 如果要建立真实投资组合，可以准备可选的券商资料，例如对账单、CSV、表格、PDF、截图或粘贴的交易记录。

你不需要学习项目脚本，也不需要手动安装技术包。Onboarding 时，助手会检查本地环境、按需安装少量依赖，并告诉你是否有需要处理的事项。

### 开始使用

1. **用 Git 安装这个 repo（推荐）。** 从 `https://github.com/lpopo0856/investment-research-agent.git` clone，然后打开文件夹。如果 Git 对你来说很陌生，请照着 [Git 新手安装指南](git-install.zh-Hans.md) 操作。如果你只是下载 ZIP，也可以先试用，但之后更新需要手动处理；长期使用建议改用 Git 安装。
2. **用你的助手打开这个文件夹。** 使用能处理这个 workspace 文件的本地助手。
3. **用普通说话方式开始。** 适合一开始这样说：

   > “请把这个 repo 切换到最新的 release tag，然后帮我开始。”
   > “创建我的第一个投资账户。”
   > “导入这份券商对账单，保存前先告诉我你读到了什么。”
   > “用 demo 数据生成组合报告。”

   建议第一句先用上面的 release tag 切换，可以确保在任何投资数据写入前已经在最新版本上。无论你用 Git 终端还是 GitHub Desktop 安装，这句话都适用。

4. **保存前先检查。** 助手在写入投资数据前，会先预览设置、交易和导入变更，让你先确认或修改方案。

## 如果你已经下载 ZIP

ZIP 下载可以用来快速试用 demo，但它是独立副本，没有 Git history，所以助手无法安全对齐 branch 或 release tag 来自动升级。

长期使用建议：

1. 在确认私人投资数据安全前，先保留你的 ZIP 副本。
2. 改用 Git 或 GitHub Desktop 重新安装 repo。
3. 如果 ZIP 副本已经包含真实投资数据，请不要删除它。用助手打开该文件夹并说：“请帮我把这个 ZIP 安装安全搬到 Git 安装。”

## 升级

如果要更新已有的本地 repo，请用助手打开这个文件夹并说：

> “请帮我安全升级这个 repo。”

助手会使用 `upgrade-management` skill：需要时先备份私人投资数据、更新 Git 安装的 repo 代码和依赖、检查账户布局，并在任何 migration 或投资数据写入前停下来请你确认。Git 安装可以安全跟随当前 branch 或 release tag；ZIP/archive 版本会被视为独立副本，助手只会说明手动更新步骤，并建议未来改用 Git 升级。

## 主要功能

你不需要懂技术细节。直接用自然语言描述意图，助手会处理后续流程。只要会改动已保存的投资数据，助手都会先列出拟变更内容，等你确认后才会保存。

- **开始使用** — 创建第一个账户、导入初始投资组合，并设置你的投资风格。
- **设置投资策略** — 定义风险承受能力、仓位大小、持有周期、语言、基准货币和不碰的领域。
- **管理多个账户** — 创建、切换、查看和合并多个券商账户、养老金账户、区域组合或策略账户。
- **记录活动** — 用普通说法新增买入、卖出、入金、出金、分红、手续费和换汇。
- **导入券商资料** — 导入对账单、导出文件、表格、PDF、截图或粘贴的交易记录。
- **修正记录** — 更正错误交易、删除重复项、核对现金，并用券商资料检查未平仓批次。
- **研究投资标的** — 按你的策略分析股票、ETF、主题、行业、市场和组合敞口。
- **每日报告** — 生成当天决策看板，包含价格、新闻、事件、警示、机会、建议调整和今日行动清单。
- **组合报告** — 查看持仓、现金、配置、盈亏、集中度、节奏和组合结构，排除日内交易噪音。
- **总报告** — 将所有账户合并成高层总览，同时保留各账户各自的数据。
- **风险和敞口检查** — 询问哪些仓位过度集中、重叠、高风险、资金不足或需要优先处理。

## 详细用法

### 询问可以做什么

如果不确定从哪里开始，可以这样说：

> “这里能做什么？”
> “给我看看有哪些功能。”
> “Help。”

助手会给你一个简单菜单，包含入门设置、记录交易、投资研究和生成报告。

### 开始使用或建立投资组合

第一次设置时，直接描述目标：

> “帮我开始。”
> “带我完成入门设置。”
> “请用这份对账单帮我建立组合。”
> “导入这份券商文件，并帮我设置投资组合。”

你可以附上券商对账单、表格、PDF、截图或粘贴的交易记录。助手会帮助创建账户、理解你的投资风格、准备记录，并确认设置可正常使用。

### 设置或查看投资风格

告诉助手你怎么投资，让后续研究按你的语气和纪律判断：

> “带我过一遍设置。”
> “查看我的投资策略。”
> “更新我的风险承受能力。”
> “把我的基准货币改成人民币。”
> “报告使用简体中文。”
> “这些行业我不碰。”

这可以包括风险承受能力、仓位大小、持有周期、入场纪律、偏好语言、基准货币和要避开的领域。

### 新增和管理账户

这里的“账户”就是分开记的账本，怎么切都行。常见分法：按家人（自己、另一半、孩子教育金）、按目标（养老、买房、应急金）、按策略（核心、卫星、试验仓）、按税务分类（一般、养老金账户），或者如果你习惯，也可以按股票市场（A 股、港股、美股、日股）切。工具不限定分法。

> “新增一个养老金账户。”
> “帮我另一半开一个账户。”
> “新增一个孩子教育金账本。”
> “新增一个卫星策略账户（高风险小仓位）。”
> “新增一个日本投资组合账户。”
> “列出我的所有账户。”
> “我现在使用哪个账户？”
> “切换到默认账户。”
> “切换到 A 股账户。”
> “生成养老金账户的报告。”
> “把所有账户合并生成总报告。”

每个账户都有自己的设置、交易、现金、持仓和报告。总报告会把账户合并成高层总览，但底层数据仍分开保存。

### 记录交易和现金流

用日常说法描述投资活动：

> “昨天 185 美元买了 30 股 NVDA。”
> “今天 400 美元卖掉 10 股 TSLA。”
> “GOOG 一季度分红 80 美元。”
> “入金 5,000 美元。”
> “为了缴税出金 1,000 美元。”
> “我付了 12 美元手续费。”
> “我把 USD 2,000 换成人民币。”

助手会解析内容、显示拟新增记录，并等你确认后才更新组合。

### 修正或核对记录

如果数据有错，直接描述要修正什么：

> “修正去年九月那笔 GOOG 批次。”
> “那笔 NVDA 应该是 20 股，不是 30 股。”
> “删除重复的分红记录。”
> “用这份对账单核对现金余额。”
> “检查我的未平仓批次是否和券商资料一致。”

助手会先说明拟修改内容，再等你确认后保存。

### 导入券商资料

附上文件后描述需求：

> “这是我的券商导出文件，请帮我导入。”
> “导入这份交易记录。”
> “导入这份 PDF 对账单，保存前先告诉我你读到了什么。”
> “这份文件有分红和交易，请加到我的账户。”

导入提示：

- 如果 PDF 有密码，请先自行打开并另存成无密码版本再导入。
- 如果文件很大，尤其是 PDF，请拆成较小批次再导入。
- 如果导入内容有歧义，助手会请你确认解读后才保存。

### 研究股票、ETF、主题或市场

可以直接问组合相关研究：

> “结合我当前组合分析 NVDA。”
> “按我的策略看，现在该买 TSM 吗？”
> “比较 TSM、NVDA、AMD。”
> “我现在 AI 敞口有多高？”
> “财报前要不要降低短线仓位？”
> “查看我的日本敞口。”
> “这周半导体我该注意什么？”

研究笔记会聚焦决策：买入、持有、减仓、避开或等待；仓位该多大；什么情况代表判断错了；接下来跟踪哪些变量。

### 生成每日报告

每日报告适合当天决策使用：

> “生成今天的每日报告。”
> “跑我的盘前报告。”
> “给我今天的组合体检。”
> “今天组合我该做什么？”

每日报告可以包含最新价格、组合健康状态、重要新闻、即将到来的事件、风险警示、高机会设置、建议调整和明确的今日行动清单。

### 生成组合报告

组合报告适合查看结构，不追逐日内噪音：

> “生成我的组合报告。”
> “显示我的配置和业绩。”
> “查看我的持仓、现金和集中度。”
> “我的组合表现如何？”

组合报告聚焦持仓、现金、配置、盈亏、集中度、投资节奏和组合结构，适合查看整本账，而不是生成当天交易建议。

### 生成跨账户总报告

有多个账户时，可以要求合并视角：

> “把所有账户合并生成总报告。”
> “显示我的整体投资组合。”
> “合并所有账户并整理我的敞口。”

总报告提供跨账户的高层总览，并避免对单一账户生成交易建议。

### 询问风险和敞口

不生成完整报告，也可以问聚焦问题：

> “我最大的集中风险是什么？”
> “我现在有多少现金？”
> “哪些仓位重叠最多？”
> “我是不是太集中在 AI 半导体？”
> “哪些持仓现在风险最高？”
> “接下来哪些事件会影响我的组合？”

### 要一份行动计划

如果只想要短清单，可以说：

> “给我今天的行动清单。”
> “我该买入、减仓，还是不动？”
> “哪些仓位要优先处理？”
> “告诉我这周要跟踪什么。”

没有明确优势时，助手会偏向不行动，而不是硬凑交易。

## 隐私

你的设置、交易记录和生成的报告都留在这个本地工作区。这个项目不会替你公开数据。你可以控制要分享哪些文件给助手，以及哪些变更要保存。

## 第三方市场数据

报告和研究可能使用公开行情和汇率来源，以及你自行提供的数据访问。数据可用性、延迟、使用限制和准确性取决于各数据提供者。

## 免责声明

仅供个人研究和记录使用，不构成投资、法律或税务建议。交易前请自行核实重要事实；决策和结果由你自行负责。
